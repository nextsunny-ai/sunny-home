가이드 = Project Knowledge에 업로드된 "Exhibition Proposal" 디자인 가이드 참조.

[주제]
본인 주제 입력 — 예: "OO전 몰입형 전시 공간 제안"

[추가 정보 — 선택]
- 전시명 (영문 대형 타이포용 + 국문) / 클라이언트 / 장소·면적 / 기간 / 담당자

[공간 데이터]
- 컨셉 한 문장 (= 2번 페이지 대형 카피):
  예) 관람객은 전시를 보지 않는다, 통과한다.
- 씬 목록 (= 씬 투어 3~4분할 · 번호·이름·1줄·면적·빛 온도):
  예) Scene 01 진입 — 달빛 게이트 · 암순응 구간 · 80㎡ · 문라이트
      Scene 02 회랑 — 빛기둥 행렬 · 헤이즈 연출 · 110㎡ · 앰버
- 도면 정보 (= 다크 라인 도면 · 축척 + 존별 면적 + 동선)
- 일정·예산·제작 범위 (= 운영 페이지 표):
  예) 01 연출 설계 2026.07—08 ₩45—60M / 제작 범위 = 공간 연출 · 미디어 콘텐츠 · 운영
- 연출컷 (있으면 파일명 · 없으면 회화적 placeholder = 어둠 + 광원 + 안개 + 관람객 실루엣)

[요구]
- 폰트 = Anton (디스플레이) + Pretendard (본문) 2종만 (모노·세리프 금지)
- 다크 베이스 #0B0B10 전 페이지 유지 · 텍스트 = 웜 화이트 #ECE9E2
- 빛 = 앰버 #E8A85C · 문라이트 #9FB4C8 2온도만 (보라·청록 네온 금지 · 글로우 떡칠 금지 — 빛은 그라데이션 면으로)
- 커버·컨셉·씬 투어 = 화면 80% 이상 연출컷 · 모든 컷에 관람객 실루엣 + 그레인 + 비네트
- 씬 투어 = 4분할 필름 프레임 (gap 4px) · 씬별 빛 온도 다르게 · 씬 번호는 Anton + 온도색
- 도면 = 다크 배경 + 절제된 화이트 라인 1~1.5px + 축척·면적·동선 점선 ("빛나는 선" 금지)
- 표·본문 = 어두운 회의실 빔 기준 가독성 (본문 13px+ · muted opacity 0.45+)
- 5장 = 커버 · 컨셉 · 씬 투어 · 공간 구성 · 운영
- 가이드의 금기 (네온·글로우·흰 페이지 혼입·이모지·영업 톤) 절대 X
- HTML 1파일 stacked · 1920×1080

출력 = HTML 코드만. 설명 없이.
