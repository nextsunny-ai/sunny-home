# Exhibition Proposal — 디자인 가이드

이 파일을 Claude 또는 ChatGPT의 **Project Knowledge**에 업로드한 뒤, 그 프로젝트에서 PPT 콘텐츠 생성을 요청하면 이 톤이 일관되게 적용된다.

몰입형 전시(미디어아트·체험전·팝업)를 클라이언트에게 제안하는 시네마틱 문법. 목표는 한 가지 — **보는 사람이 "이 공간에 들어가 보고 싶다"고 느끼게 하는 것.** 연출컷이 분위기를 팔고, 도면·표가 실행력을 증명한다. 페이지의 앞쪽은 어둠과 빛, 뒤쪽은 숫자다.

---

## 폰트

- **디스플레이** = Anton (컨덴스드 대문자 — 전시명·씬 번호·존 번호)
- **본문·표** = Pretendard Variable (400 / 500 / 600 / 700)
- 모노스페이스·세리프 = 사용 금지
- system-ui / Inter / Roboto 사용 금지
- Google Fonts (Anton) + Pretendard CDN `@import` 사용

## 컬러 — 빛 2온도 시스템

```
배경        #0B0B10  (다크 — 순흑 #000 금지)
배경 외곽   #060609
텍스트      #ECE9E2  (웜 화이트 — 순백 금지)
앰버 (난색)   #E8A85C  — rgba(232, 168, 92, α)
문라이트 (한색) #9FB4C8  — rgba(159, 180, 200, α)
muted       rgba(236, 233, 226, 0.45 ~ 0.7)
구분선      rgba(236, 233, 226, 0.14 ~ 0.3) · 강조선 0.5
도면 선     rgba(236, 233, 226, 0.8) — 1~1.5px 화이트 라인
```

- **빛은 앰버·문라이트 2온도뿐.** 보라·핑크·청록 네온 금지. 무지개 씬 컬러 금지
- 빛 표현 = 그라데이션 면(샤프트·풀·할로)으로만. **box-shadow·blur 글로우 떡칠 금지**
- 한 씬 = 한 온도가 지배 (앰버 씬 / 문라이트 씬). 두 온도가 한 씬에서 같은 비중으로 섞이면 탁해진다

## 이미지(연출컷) 처리 룰 (= 이 스타일의 핵심)

1. **지면 배분** — 커버·컨셉·씬 투어 = 화면의 80% 이상이 연출컷. 텍스트는 라벨·전시명·한 줄 설명까지만. 도면·운영 페이지만 정보 우선
2. **연출컷 문법** — 어둠 속 광원(빛기둥·포털·미디어 면·문라이트 오브제) + 안개 + 관람객 실루엣 2~5명. 실루엣이 스케일과 "들어가 보고 싶다"를 만든다 — 빈 공간 컷 금지
3. **빛 온도 통일** — 씬별 온도를 정하고(예: 진입 = 문라이트, 회랑 = 앰버) 도면·존 리스트의 번호 색까지 동일 온도로 연동한다
4. **렌더·레퍼런스 혼용** — 같은 페이지에는 같은 어둠 기준(배경 명도)으로 보정한 컷만. 밝은 레퍼런스 사진을 쓸 땐 다크 톤으로 보정하거나 별도 페이지로 분리
5. **placeholder** — 실제 렌더가 없으면 다층 그라데이션 + clip-path로 회화적으로 구성 (빛기둥 홀·회랑 포털·미디어 면·문라이트 게이트). 빈 단색 박스·빗금 박스 금지
6. **필름 그레인 + 비네트** — 모든 연출컷에 저강도 그레인과 비네트. 평평한 그라데이션 면이 그대로 보이면 실패

## 다크 화면 가독성 (= 발표 환경 룰)

발표 환경 = 어두운 회의실 빔프로젝터 기준. 빔은 명암 대비가 깎인다:

- 본문 최소 13px · 표 본문 14px 이상 · muted 텍스트도 opacity 0.45 미만 금지
- 정보 텍스트는 웜 화이트 — 회색(#888 류) 본문 금지
- 도면 = 다크 배경 + 화이트 라인 1~1.5px + 옅은 모눈. "빛나는 선" 연출 금지 — 절제된 라인이 실무 신뢰를 만든다
- 표 구분선 = rgba 화이트 0.14 이상 · 헤더 행 0.5 강조선
- 어두운 사진 위 텍스트 = 하단 어둠 영역(floorline 그라데이션) 위에만

## 레이아웃

- 슬라이드 = **1920 × 1080**
- 러닝 헤드 = 내지 상단 56px: 좌 `전시명 — Proposal` / 우 `NN · 섹션명` (번호만 앰버) + 하단 1px 라인
- 커버 = 풀블리드 연출컷 + 좌하단 전시명 (Anton 대형) + 우하단 메타 (Client·Venue·Period)
- 씬 투어 = 세로 4분할 (gap 4px — 필름 프레임 감각). 각 분할 = 연출컷 + 하단 설명 블록 (씬 번호·이름·1줄·면적)
- 도면 페이지 = 도면 7 : 존 리스트 5
- 칩·둥근 카드·그림자 박스 = 사용 금지

## 타이포

| 요소 | 폰트 | 크기 | 비고 |
|---|---|---|---|
| 전시명 (커버) | Anton | 150~190px | uppercase · line-height 0.92 |
| 컨셉 한 문장 | Pretendard 700 | 42~48px | line-height 1.4 |
| 섹션 타이틀 | Pretendard 700 | 36~40px | + 영문 캡스 서브 (0.28em) |
| 씬 번호 | Anton | 13px | letter-spacing 0.18em · 온도색 |
| 러닝 헤드·라벨 | Pretendard 600 | 12~13px | uppercase · 0.2~0.3em |
| 본문·표 | Pretendard 400~700 | 13~15px | 표 숫자·면적 단위 통일 |

## 슬라이드 5장 패턴

1. **커버 (Cover)** — 빛기둥 홀 풀블리드 + 관람객 실루엣 + 전시명 대형 타이포 + 메타 3종
2. **컨셉 (Concept)** — 시네마틱 와이드 1컷 (회랑 끝 포털 등) + 한 문장 컨셉 + 보조 2줄
3. **씬 투어 (Scene Tour)** — 씬별 연출컷 3~4분할, 씬마다 빛 온도 다르게 + 씬 번호·이름·1줄·면적
4. **공간 구성 (Floor Plan)** — 다크 라인 도면 (축척 표기 · 존 번호 온도색 · 동선 점선) + 존 리스트 + 면적 합계
5. **운영 (Production)** — 일정·예산 표 + 제작 범위 3컬럼 (공간 연출 / 미디어 콘텐츠 / 운영) + 연락처

실제 제안서는 씬 수만큼 패턴 2·3을 반복 확장한다 (씬당 와이드 1 + 디테일 1).

## 금기 (절대 X)

- 보라·핑크·청록 네온 / 빛 3온도 이상 / 무지개 씬 컬러
- box-shadow·blur 글로우 떡칠 (빛은 그라데이션 면으로)
- 흰 배경 페이지 혼입 (전 페이지 다크 유지)
- 빈 공간 연출컷 (실루엣 없는 컷)
- "빛나는 선" 도면 · 도면 정보(축척·면적) 누락
- 모노스페이스·세리프 폰트 · 칩 · 둥근 카드
- 이모지
- 영업·홍보·광고 톤 문구 ("Powerful·Premium·혁신적·압도적·새로운 ___을 만나다")
- AI 슬롭 패턴 (보라→핑크 그라디언트 · 글로우 · 카드 떡칠)
- muted 텍스트 opacity 0.45 미만 (빔 환경에서 사라진다)

## 출력 형식

- HTML = `<section class="slide">` vertically stacked · CSS `<style>` 임베디드
- 폰트 = Google Fonts `@import` (Anton) + Pretendard CDN
- 실제 렌더 삽입 = `<img>` / `background-image` + `object-fit: cover` + 위에 그레인·비네트 레이어 유지
- 면적·일정·예산 = 실제 값 우선. 더미는 더미임을 표시
- 콘텐츠(전시명·씬 구성)는 사용자 주제대로
