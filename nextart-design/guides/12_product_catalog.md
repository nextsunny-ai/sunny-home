# Product Catalog — 디자인 가이드

이 파일을 Claude 또는 ChatGPT의 **Project Knowledge**에 업로드한 뒤, 그 프로젝트에서 PPT 콘텐츠 생성을 요청하면 이 톤이 일관되게 적용된다.

MUJI류 미니멀 제품 카탈로그의 문법 — **장식을 전부 걷어내고 제품·인덱스 번호·스펙만 남긴다.** 액센트 컬러도 없다. 위계는 색이 아니라 모듈 크기(피처드 셀)와 타이포 굵기로 만든다. 바이어·입점처 제출 기준 — 품번·가격·치수·소재·MOQ까지 채워야 완성이다.

---

## 폰트

- **라틴·숫자·스펙** = Manrope (400 / 500 / 600 / 800 · 숫자는 `font-variant-numeric: tabular-nums`)
- **한국어** = Pretendard Variable (제품명·본문)
- 모노스페이스·세리프 = 사용 금지
- system-ui / Inter / Roboto 사용 금지
- Google Fonts + Pretendard CDN `@import` 사용

## 컬러

```
배경        #F4F4F1
제품 셀     #EBEBE7  (전 셀 동일)
잉크        #26262A
오브제 톤   #9C9A93 → #6A6862  (웜그레이 음영)
muted       rgba(38, 38, 42, 0.35~0.65)
```

- **액센트 컬러 없음** — 가격·강조 모두 잉크색. 위계 = 모듈 크기와 weight 차이로
- **`#FFFFFF` / `#000000` 절대 X**
- 컬러가 들어가는 유일한 곳 = 제품 사진 자체

## 이미지 처리 룰 (= 이 스타일의 핵심)

1. **배경 통일** — 모든 제품컷은 누끼 또는 동일 단색 셀 배경 `#EBEBE7` 위. 누끼·연출컷 혼용 X (연출컷은 별도 챕터 페이지에만)
2. **크기 비례** — 같은 모듈 그리드 안에서는 제품 간 실제 크기 비례 유지. 키링이 의자보다 크게 보이면 안 된다
3. **그림자 통일** — 바닥 그림자는 전 제품 같은 위치(하단 중앙)·같은 강도의 소프트 타원. 안 쓰면 전부 안 쓴다
4. **셀 여백** — 제품은 셀 면적의 60~75%. 셀 가장자리에 닿지 않게
5. **피처드 모듈** — 페이지당 1개 제품만 2×2 피처드 셀로 키우고 스펙 라인(치수·소재·MOQ)을 추가한다. 나머지는 1×1 균등
6. **이미지 비율** — 셀 = 그리드가 결정 (비율 고정 X) · 제품은 셀 중앙 정렬
7. **placeholder** — 제품 사진이 없으면 웜그레이 음영의 오브제 실루엣(CSS 도형)으로 형태·비례를 잡고 품번은 실제 값. 타사 제품 사진 임시 사용 절대 X

## 레이아웃

- 슬라이드 = **1920 × 1080**
- 상단 마스트헤드 행 (128px) = 좌측 `Product Index 2026` (Manrope 800 · 연도는 muted) / 우측 메타 2줄 (Vol · 품목 수 · 통화)
- 모듈 그리드 = **4컬럼 × 2행 · gap 12px** — 피처드 1개(2×2) + 1×1 셀들
- 셀 캡션 행 = 셀 아래 1줄: `인덱스 번호(800·muted) · 제품명(Pretendard 600) + 품번(small muted) · 가격(tabular·우측 정렬)`
- 칩·hairline 바·모노 라벨·둥근 카드 = 사용 금지

## 타이포

| 요소 | 폰트 | 크기 | 비고 |
|---|---|---|---|
| 마스트헤드 | Manrope 800 | 52~64px | letter-spacing -0.045em |
| 마스트헤드 메타 | Manrope 600 | 13px | 우측 정렬 · muted |
| 인덱스 번호 | Manrope 800 | 13px | muted 0.35 |
| 제품명 | Pretendard 600 | 14~15px | 품번은 Manrope 500 11.5px muted |
| 가격 | Manrope 600 | 13~14px | tabular-nums · 우측 정렬 |
| 스펙 라인 | Manrope 500 | 11.5~12px | `W × D × H mm` · 소재 · MOQ |

## 데이터 표기 규칙

- 품번 = `GM-2601` 형식 (브랜드 약어 + 연도 2자리 + 일련번호)
- 가격 = `₩240,000` (콤마 · VAT 포함 여부는 마스트헤드 메타에 1회 표기)
- 치수 = `W 620 × D 640 × H 750 mm`
- 인덱스 번호 = 페이지 내 `01`부터, 카탈로그 전체는 품번으로 추적
- 카탈로그 유효기간 = 마지막 페이지(주문 안내)에 표기

## 페이지 5장 패턴

1. **인덱스 (Index)** — 마스트헤드 + 모듈 그리드 (피처드 1 + 1×1 4~5개), 셀마다 캡션 행
2. **카테고리 (Category)** — 1×1 균등 8셀 (4×2) + 카테고리 타이틀 행
3. **피처드 상세 (Featured)** — 좌 대형 제품컷 / 우 스펙 시트 (치수·소재·구성·원산지·출시·가격·MOQ — Manrope 표)
4. **가격표 (Price List)** — 전 SKU 표: 품번·제품명·치수·소비자가·박스 입수·MOQ (tabular 숫자 우측 정렬)
5. **주문 안내 (Ordering)** — 발주 절차 3단계 + 담당·메일·납기 + 유효기간

실제 카탈로그는 카테고리 수만큼 패턴 2·3을 반복 확장한다.

## 금기 (절대 X)

- 액센트 컬러 (모노크롬 유지 — 컬러는 제품 사진만)
- 제품컷 배경 혼용 (셀 배경 #EBEBE7 통일)
- 가격 없는 제품 (미정 = "가격 미정 · 출시 시 공지" 명시)
- 모노스페이스·세리프 폰트 · 칩 · hairline 메타 바
- 둥근 카드 · 그림자 박스 · 그라디언트
- 이모지
- 영업·홍보·광고 톤 문구 ("Powerful·Premium·혁신적·새로운 ___을 만나다")
- AI 슬롭 패턴 (보라→핑크 그라디언트 · 글로우)
- 타사 제품 사진 placeholder

## 출력 형식

- HTML = `<section class="slide">` vertically stacked · CSS `<style>` 임베디드
- 폰트 = Google Fonts `@import` (Manrope) + Pretendard CDN
- 품번·가격·치수 = 사용자 제공 값 우선. 더미는 일관된 체계로
- 콘텐츠(브랜드·품목)는 사용자 주제대로
